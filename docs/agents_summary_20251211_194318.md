# agents目录文件功能说明

## 1. test_openaiagentsdk_doubao.py
**功能**: 豆包与OpenAI Agent SDK集成的最简示例。
**关键内容**: 
- 配置豆包Ark API的基础URL、模型ID和API Key；
- 设置兼容OpenAI的AsyncOpenAI客户端，替换Agent SDK的默认客户端；
- 定义一个简单的Agent（Assistant）并运行测试对话；
- 演示豆包服务通过OpenAI SDK接入的基础流程。

## 2. test_oasdk_db_file_io.py
**功能**: 多Agent协作的项目文件分析与文档生成系统。
**关键内容**: 
- **工具定义**: 提供`list_python_files`（列Python文件）、`read_file`（读文件）、`write_text_file`（写总结文件）、`image_to_data_url`（图片转Data URL）等工具；
- **Agent体系**: 
  - FileTreeAgent: 负责列出指定目录下的Python文件；
  - FileReaderAgent: 读取文件并总结功能；
  - WriterAgent: 将总结写入带时间戳的Markdown文件；
  - CoordinatorAgent: 协调上述Agent完成全流程；
- **封装函数**: `run_repo_summary`启动文档生成任务，支持指定分析目录、输出路径和任务描述。

## 3. test_openaiagentsdk_doubao_vision.py
**功能**: 豆包多模态视觉问答示例（图片+文本查询）。
**关键内容**: 
- 配置豆包Ark API参数和本地图片路径、查询问题；
- 实现`to_data_url`函数将本地图片转为Data URL；
- 构造多模态输入（图片+文本问题），通过豆包兼容接口执行视觉问答；
- 演示豆包处理图片内容并生成回答的能力。
